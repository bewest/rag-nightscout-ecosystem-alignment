Predictive-Signal Validation (Candidly's "AUC-proof" step)

Leave-patient-out cross-validated AUC for predicting whether the next turn resolves well: glycemic-only baseline = 0.638, full (with physiology features) = 0.615 (delta = -0.022). In this first cut, the physiology feature set did not yet improve on the simple glycemic-only baseline. This is an honest "not yet" finding, not a failure: it means the current mean-aggregated per-turn physiology features (activity, flux/EGP, saturation, glycogen proxy, site wear) don't yet add forward-predictive value beyond current TIR/TBR/CV continuity. Top physiology features by (non-cross-validated) importance: bolus_active_row_count (+0.52), smb_active_row_count (-0.48), meal_count (-0.33).

Controller-Lineage Stratification

21 patients have known controller lineage (EXP-2753: Loop vs Trio/oref1). Mean TIR differs substantially by controller at the population level (see figure below), but adding controller identity to the leave-patient-out classifier changed within-patient AUC by only +0.005 — because a patient-level-constant covariate cannot discriminate between that same patient's own turns. This guards against Simpson's-paradox-style confounding: the population difference is real, but it does not appear to be silently driving the predictive-validation result above.
