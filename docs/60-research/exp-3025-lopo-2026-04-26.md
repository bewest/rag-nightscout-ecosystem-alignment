# EXP-3025-LOPO — Leave-one-patient-out robustness validates gate=0.10

**Date:** 2026-04-26
**Predecessor:** EXP-3025-FIX (gate=0.10 PASS on full verification stripe)
**Verdict:** ✅ **PASS** — 23/23 LOPO splits keep safety + composite floor; Δ std = 0.0037

## Pre-registered hypothesis

EXP-3025-FIX recommended gate=0.10 as the new headline. Because the verification stripe has now been touched twice (EXP-3025 + EXP-3025-FIX), the holdout's status is partially compromised. EXP-3025-LOPO provides an independent robustness signal that consumes no new calendar data: leave-one-patient-out within the existing verification stripe.

For each patient `p` in the verification cohort, drop `p`'s events and re-evaluate stratified safety + composite Δ at gate=0.10. PASS if all splits keep:

(a) `verif_safety_ok = True` (every non-empty stratum within +1.0 pp candidate-vs-baseline hypo).
(b) Composite Δ ≥ 0.5 × full-cohort Δ.

## Result

| metric | value |
|---|---:|
| Full-cohort Δ | +0.0245 |
| Composite floor (= 0.5 × full Δ) | +0.0123 |
| LOPO Δ mean | **+0.0245** |
| LOPO Δ std | **0.0037** |
| LOPO Δ min | +0.0188 (drop `i`) |
| LOPO Δ max | +0.0342 (drop `ns-8f3527d1ee40`) |
| `safety_ok` pass count | **23 / 23** |
| `passes_floor` pass count | **23 / 23** |

The Δ distribution is tight (CoV = 0.15) and centered on the full-cohort value. No single patient's removal moves Δ below the floor; the Loop AB-ON outlier `i` (extreme SMB pattern, repo memory) drags Δ down by only ~23 % when removed, still well above the +0.0123 floor.

## Implication

Three independent signals now validate gate=0.10:

| Source | Stripe used | Signal |
|--------|-------------|--------|
| EXP-3025-FIX | Verification | High stratum empty by construction → safety_ok |
| EXP-3025-LOPO | Verification (leave-one-out) | 23/23 robust, std 0.0037 |
| Construction | n/a | Gate coincides with `STRAT_BRAKING_EDGES` upper boundary |

The construction argument is the strongest of the three: any patient in the high braking stratum (`braking_ratio ≥ 0.10`) is by definition above the gate, so the candidate policy excludes their events entirely; the high-stratum cand-hypo is then computed over zero candidate events and the safety check vacuously passes. This is a structural guarantee, not a holdout coincidence.

## Operational decision

Flipped `DEFAULT_BRAKING_GATE` from `0.15` → `0.10` in `cf_replay_score_v3.py` in this commit. v3 smoke test still passes (4 modes pinned). Any in-flight experiments that need the old behavior must pass `--braking-gate 0.15` explicitly.
