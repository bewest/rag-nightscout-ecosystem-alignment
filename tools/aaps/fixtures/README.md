# AAPS-shape profile fixtures

Minimal but realistic JSON bodies that mirror what AAPS actually sends to Nightscout when the user
saves the Local Profile editor or activates a profile-switch. All fixtures use deterministic
timestamps so tests are reproducible.

| File | Channel | Source | Reference |
|---|---|---|---|
| `v1-profile-store-initial.json` | V1 socket `dbAdd("profile", …)` | `DataSyncSelectorV1.processChangedProfileStore:786-805` | First-ever store push |
| `v1-profile-store-edit.json` | V1 socket `dbAdd("profile", …)` | same | Subsequent edit (later `startDate`) |
| `v3-profile-store-initial.json` | V3 REST `POST /v3/profile` | `NSAndroidClientImpl.createProfileStore:436` adds `app:"AAPS"` | First-ever store push |
| `v3-profile-store-edit.json` | V3 REST `POST /v3/profile` | same | Subsequent edit |
| `v1-treatment-profile-switch.json` | V1 socket `dbAdd("treatments", …)` | `ProfileSwitchExt.toJson` | A profile-switch with embedded `profileJson` (what produces `Test@@@@@<mills>` columns) |

Bodies are generated by `tools/aaps/build-fixtures.js` and consumed by:
- `externals/cgm-remote-monitor/tests/fixtures/aaps/*.json` (copies for c-r-m roundtrip tests)
- `tools/aaps/dropdown-logic-probe.js`
